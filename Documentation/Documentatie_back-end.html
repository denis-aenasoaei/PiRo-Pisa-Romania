Enuntul problemei:

"Pe baza unui API REST/GraphQL propriu, sa se implementeze o aplicatie Web de vizualizare flexibila a datelor referitoare la rezultatele obtinute de elevii romani la testul Pisa 2018. Statisticile si vizualizarile generate – minim 3 maniere – vor putea fi exportate in formatele CSV, PNG si SVG. Sistemul va oferi suport si pentru cautarea/filtrarea multi-criteriala a datelor, inclusiv comparatii cu alte tari – a se utiliza Education Statistics."

Tehnologii folosite

Tehnologiile folosite pentru realizarea backend-ului sunt cele din suita XAMPP (Cross-Platform, Apache, MariaDB(MySQL), PHP, Perl(nefolosit))

  1. PHP pentru server si procesarea cererilor
  2. MySQL ca mediu de date persistente

Descrierea aplicatiei

Arhitectura serverului:

Server-ul este implementat in PHP, cu un stil de programare orientat obiect.

Punctul de intrare al aplicatiei este pagina index.php, care ruteaza cererea primita pe baza datelor prezente in aceasta. Rutele invalide sunt redirectionate catre pagina de pornire.

Aplicatia implementeaza modelul MVC de organizare, deoarece acest mod de organizare ofera un grad ridicat de separare a rolurilor (Separation of concerns).

Aplicatia ofera 3 pagini pe care un utilizator obisnuit le poate accesa, alaturi de inca doua care serverc unui mod de administrare a datelor prezente, care necesita o logare in prealabil

  • Paginile disponibile utilizatorului obisnuit:
    1. Home, punctul de plecare al aplicatiei, ofera informatii generale despre PISA si ce misiune are aceasta organizatie
    2. Contact, de unde un utilizator poate lasa un comentariu care poate fi raspuns de cineva care are acces la pagina de administrator
    3. Results, de unde utilizatorul poate vedea rezultatele testului PISA, prezentate sub forma de grafic
  • Paginile de administrare:
    1. Log-in, necesar accesarii paginii de administrator
    2. Admin, de unde se pot vedea si modifica in totalitate datele din BD
